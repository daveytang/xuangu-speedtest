# [OPEN] Debug Session: speedtest-site-audit

## Goal
- 全面梳理测速网站功能模块
- 复现并记录核心测速功能异常
- 排查 UI/交互/响应式问题
- 基于运行时证据修复问题并验证

## Initial Hypotheses
1. 前端测速流程与真实测速实现的衔接不完整，导致结果输出异常或失真。
2. 指标计算逻辑存在单位换算、采样或边界值处理问题。
3. 异步状态管理存在竞争条件，影响测速流程稳定性。
4. UI 组件层级与断点样式设计不合理，导致响应式与交互体验问题。
5. 构建配置或浏览器兼容性不足，导致运行环境差异问题。

## Problem Statement
- 实际表现：核心测速流程在下载/上传阶段容易失败，页面缺少丢包率模块，中止测速不会稳定生效，UI 存在遮挡与信息层级问题。
- 期望表现：下载、上传、延迟、抖动、丢包率完整输出；中止行为可控；桌面与移动端布局清晰稳定。
